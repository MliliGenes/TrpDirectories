THE HOOK

What is this?
enum in C++98 is a set of integral constants grouped under one type, while enum class (introduced in C++11) is a scoped, strongly typed version that avoids implicit conversions and name collisions.

Why care?
Enums structure symbolic constants in parsers, state machines, and tokenizers. enum class provides type safety and clarity, but in C++98 you must rely on classic enums with careful design.

Mental model:
Think of old enum as sticky notes scattered on a wall (easy but messy). enum class is like putting those notes into clearly labeled folders (organized and safe).

⚙️

HOW IT WORKS

Core mechanics

  • Classic enum: Weakly typed, values injected into outer scope, implicitly convertible to int.
  • Enum class: Scoped to their type, no implicit conversion to int, comparisons only within the same enum.

Governing principles

  • C++98 → only old enums available.
  • C++11+ → enum classes provide safer scoping and typing.
  • Trade-off: old enums are concise but error-prone; enum classes are verbose but safer.

⚠️

AVOID THESE

  • Mixing ints and enums → Avoid using if (token == 0), use enum names instead.
  • Name collisions → Different enums using the same value like STRING can conflict.
  • Assuming enum size → Underlying type is compiler-defined in C++98.
  • Using enums as bitmasks carelessly → Requires manual operator overloads.
